## Synthesis: PERCEIVE -> ANALYZE -> VALIDATE -> ACT

### PERCEIVE (What we observe)

**External signals:** The site appears to have zero indexed pages in Google. Brand searches for "kicksnare" return music results (bands, SoundCloud). No robots.txt, no sitemap.xml, no structured data, no OG tags. The domain has no measurable authority or external citations.

**Internal signals:** Strong conversion-focused copywriting with specific metrics (2.6x demo bookings, 3.1x ROAS, 41% bounce reduction). ~567 words of meaningful content. Three named clients. No named individuals behind the agency. No privacy policy (legally required in UK). Booking link points to a Vercel staging domain. Primary contact method is a Twitter DM.

**User experience signals:** 20px horizontal overflow on mobile. Hamburger menu below minimum touch targets. 34 DOM elements start at `opacity:0` requiring JS to become visible. Contact section buried at 96% scroll depth on mobile. Large blank mid-page sections from scroll-triggered animations.

### ANALYZE (What it means)

**First principle (THINK):** This site is structurally invisible to search engines. It presents a single URL with ~567 words to compete for queries where Google rewards deep, multi-page service sites, directories, and editorial listicles with 3,000-8,000 words per page. The page-type mismatch is not an optimization gap -- it's an architectural mismatch with what Google has decided this category requires.

**Lateral connection (CONNECT-lateral):** The site's actual function is a referral-traffic landing page, not an organic discovery asset. It scores highest for the referral visitor persona (79/100) and lowest for the marketing manager (55/100) -- the persona most likely to arrive via organic search. The copywriting quality is genuinely above average; the problem is not content quality but content architecture and discoverability infrastructure.

**System connection (CONNECT-system):** Every finding connects back to one root cause: the site was built for conversion of warm traffic, not for discovery. Fixing individual SEO issues (adding a robots.txt, canonical tag, etc.) without addressing the single-page architecture would produce marginal improvement. The fixes form a dependency chain: indexation infrastructure first, then content architecture, then on-page optimization.

### VALIDATE (Is this real?)

**Falsifiability check (ACCEPT):** The "zero indexed pages" finding should be verified via Google Search Console. If GSC shows indexed pages with impressions, the organic visibility diagnosis changes significantly. The page-type mismatch finding can be validated by searching the target queries and confirming the SERP composition.

**What could invalidate these recommendations:** If the business model is 100% referral/paid-traffic with no intent to pursue organic, most recommendations below Priority 3 become optional. The legal compliance items (privacy policy, company identity) remain mandatory regardless.

## Findings by Priority

### CRITICAL (Blocks indexing or causes penalties)

**1. No robots.txt** (404)
- First principle: Search engines and AI crawlers have no directives for crawl behavior
- Dependency: Required before sitemap, IndexNow, or AI crawler management
- How we'd know this failed: Crawl stats in GSC show no improvement after adding
- Leading indicator: GSC "Pages" report shows discovered URLs

**2. No sitemap.xml** (404)
- First principle: Single-page site with only one URL, but sitemap signals intentional crawl management
- Dependency: Unblocks sitemap submission in GSC
- How we'd know this failed: GSC sitemap report shows errors
- Leading indicator: "Submitted and indexed" count in GSC

**3. No canonical URL**
- First principle: Without canonical, the same page may be indexed under multiple URL variants (www vs non-www, trailing slash, query parameters)
- Dependency: Must be set before any content expansion
- How we'd know this failed: Duplicate URL entries in GSC
- Leading indicator: Single URL per page in "Pages" report

**4. H1 contains duplicated text from animation**
- The rotating word animation injects text twice in the DOM: *"We build the growth tools, web apps, landing pages, and digital products**growth toolsweb appslanding pagesdigital products** you need to grow."*
- First principle: Googlebot reads the raw DOM; this H1 is garbled
- Dependency: None
- How we'd know this failed: H1 still shows duplicated text in GSC's rendered HTML
- Leading indicator: Clean H1 in "URL Inspection" tool

**5. Missing privacy policy** (UK legal requirement)
- Under GDPR and the Companies Act, a UK business website collecting contact information must display a privacy policy, company name, and registered address
- First principle: Legal compliance, not optional SEO
- Dependency: None
- How we'd know this failed: ICO complaint or quality rater downgrade
- Leading indicator: Policy page exists and is linked from footer

**6. 20px horizontal overflow on mobile**
- Content bleeds past the 375px viewport, causing horizontal scroll
- First principle: Fails Google's mobile-friendliness criteria
- Dependency: None
- How we'd know this failed: Mobile Usability report in GSC still shows errors
- Leading indicator: `scrollWidth === clientWidth` on mobile viewports

### HIGH (Significantly impacts rankings -- fix within 1 week)

**7. Zero structured data**
- No JSON-LD, Microdata, or RDFa of any kind. Zero schema.org references.
- Recommended schema: `Organization`, `ProfessionalService`, `WebSite`, `Person` (once founder is named)
- First principle: Rich results eligibility requires structured data
- Dependency: Naming a founder (Priority 8) enables Person schema
- How we'd know this failed: Rich Results Test shows validation errors
- Leading indicator: Rich result impressions in GSC Performance report

**8. No named individual / founder**
- All content is voice-of-brand with no human identity. "7 years proven" is attributed to an unnamed team.
- First principle: E-E-A-T requires verifiable human expertise signals
- Dependency: Unblocks Person schema, author bylines, LinkedIn authority signals
- How we'd know this failed: No improvement in AI citation or authority metrics
- Leading indicator: Named entity appears in Knowledge Panel or AI Overviews

**9. No Open Graph / Twitter Card meta tags**
- Social shares render with no controlled preview image, title, or description
- First principle: Social sharing is a discovery and authority signal
- Dependency: None
- How we'd know this failed: Social shares still show blank previews
- Leading indicator: OG meta tags visible in page source

**10. Single-page architecture limits organic potential**
- Every target query rewards multi-page sites with dedicated service pages, case studies, and blog content
- First principle: A single URL cannot target multiple keyword clusters
- Dependency: This is a strategic decision, not a quick fix
- How we'd know this failed: No ranking improvement after 6 months of other fixes
- Leading indicator: Multiple URLs indexed in GSC

**11. Hamburger menu touch target below 48px minimum**
- Measures 38x32px on tablet/mobile -- fails WCAG 2.5.5 and platform HIG
- First principle: Primary navigation must be accessible
- Dependency: None
- How we'd know this failed: Mobile usability issues persist
- Leading indicator: Touch target passes 48x48px in dev tools

**12. "DM on X" as primary contact -- high friction**
- Professional B2B buyers avoid social media DMs for business enquiries
- First principle: Conversion path must match buyer expectations
- Dependency: None
- How we'd know this failed: Contact form/calendar submissions don't increase
- Leading indicator: Multiple low-friction contact options visible on page

### MEDIUM (Fix within 1 month)

**13. No llms.txt file**
- Missing the emerging standard for AI-readable site summaries
- First principle: Direct AI retrieval target bypasses rendering issues

**14. Case studies too brief and gated**
- 30-40 words each, "full case studies on request" -- treats best trust asset as gated content
- First principle: Trust content should be freely accessible

**15. Booking link points to staging domain**
- `proj-astro-seven.vercel.app` undermines credibility for any visitor who inspects links
- First principle: All external links should resolve to professional, branded URLs

**16. 34 elements start at opacity:0**
- Content requires JS execution to become visible to users (crawlers see it, but LCP is impacted)
- First principle: Core content should be visible without JS for optimal CWV

**17. Carousel/slider dots at 20x20px**
- 58% of minimum touch target size, affecting 8 interactive elements
- First principle: All interactive elements must meet 48x48px minimum

**18. No favicon or app icons detected**
- Missing browser tab icon and mobile bookmark icon
- First principle: Basic brand presence in browser chrome

**19. Empty image alt text**
- At least one content image has `alt=""` -- fails WCAG 1.1.1 if not purely decorative
- First principle: Non-decorative images need descriptive alt text

### LOW (Backlog)

**20. No blog or long-form content** -- Zero organic content surface
**21. No third-party directory profiles** -- Clutch, DesignRush own agency SERPs
**22. Brand confusion** -- "kicksnare" returns music results, not web design
**23. No IndexNow protocol** -- Would speed re-indexation after changes
**24. Font loading** -- 3 WOFF2 fonts preloaded (good), but 30 TTF files in /fonts/ directory
**25. Contact section at 96% scroll depth on mobile** -- Consider sticky CTA
